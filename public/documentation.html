API Documentation

This documentation provides details about the API endpoints available in this application.

1. Root Endpoint

Description: Returns a welcome message when accessing the root of the application.

Endpoint URL: /

Method Type: GET

Query Parameters: None

Request Body: None

Response Format:

Welcome to my app!

2. Add New User

Description: Creates a new user and adds them to the list of users.

Endpoint URL: /users

Method Type: POST

Request Body:

{
  "name": "John Doe",
  "favoriteMovies": []
}

Response Format: Returns the newly created user object.

{
  "id": "some-uuid",
  "name": "John Doe",
  "favoriteMovies": []
}

3. Update User Information

Description: Updates the name of an existing user by ID.

Endpoint URL: /users/:id

Method Type: PUT

Query Parameters: id (User ID)

Request Body:

{
  "name": "Jane Doe"
}

Response Format: Returns the updated user object.

{
  "id": "some-uuid",
  "name": "Jane Doe",
  "favoriteMovies": []
}

4. Add Movie to User's Favorites

Description: Adds a movie title to a user's list of favorite movies by user ID.

Endpoint URL: /users/:id/:movieTitle

Method Type: POST

Query Parameters: id (User ID), movieTitle (Movie Title)

Request Body: None

Response Format: Confirmation message indicating the movie was added.

" (movieTitle) has been added to user (id)'s array"

5. Remove Movie from User's Favorites

Description: Removes a movie title from a user's list of favorite movies by user ID.

Endpoint URL: /users/:id/:movieTitle

Method Type: DELETE

Query Parameters: id (User ID), movieTitle (Movie Title)

Request Body: None

Response Format: Confirmation message indicating the movie was removed.

"(movieTitle) has been removed from user (id)'s array"

6. Delete User

Description: Deletes a user from the list by ID.

Endpoint URL: /users/:id

Method Type: DELETE

Query Parameters: id (User ID)

Request Body: None

Response Format: Confirmation message indicating the user was deleted.

"User (id) has been deleted"

7. Get All Movies

Description: Returns a list of all movies.

Endpoint URL: /movies

Method Type: GET

Query Parameters: None

Request Body: None

Response Format: A JSON array of movie objects.

[
  {
    "Title": "Movie Title",
    "Genre": { "Name": "Genre Name", "Description": "Genre Description" },
    "Director": { "Name": "Director Name", "Bio": "Director Bio", "Birth": "Director Birth Date" },
    // other movie details...
  },
  // other movies...
]

8. Get Movie by Title

Description: Returns data about a single movie by title.

Endpoint URL: /movies/:title

Method Type: GET

Query Parameters: title (Movie Title)

Request Body: None

Response Format: A JSON object with the movie details.

{
  "Title": "Movie Title",
  "Genre": { "Name": "Genre Name", "Description": "Genre Description" },
  "Director": { "Name": "Director Name", "Bio": "Director Bio", "Birth": "Director Birth Date" },
  // other movie details...
}

9. Get Genre by Name

Description: Returns the details of a genre by its name.

Endpoint URL: /movies/genre/:genreName

Method Type: GET

Query Parameters: genreName (Genre Name)

Request Body: None

Response Format: A JSON object with the genre details.

{
  "Name": "Genre Name",
  "Description": "Genre Description"
}

10. Get Director by Name

Description: Returns the details of a director by their name.

Endpoint URL: /movies/directors/:directorName

Method Type: GET

Query Parameters: directorName (Director Name)

Request Body: None

Response Format: A JSON object with the director details.

{
  "Name": "Director Name",
  "Bio": "Director Bio",
  "Birth": "Director Birth Date",
  // other director details...
}
